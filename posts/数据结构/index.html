  • 算法和算法分析

    • 概念
    • 问题：需要解决的问题

    • 有穷性

    • 算法分析
    • 基本假设
      • 空间单位
      • 时间单位
    • 算法代价看作实例规模的函数
      • 时间代价
      • 空间代价
    • 分情况考虑代价
      • 最好（无价值）
      • 最坏情况的时间代价
      • 平均时间代价
    • 算法复杂度
      • 渐近函数
      • 时间复杂度
      • 常用：$O(1)常量复杂度\O(\log n)对数复杂度\O(n)线性复杂度\O(n\log n)\O(n^2)平方复杂度\O(n^3)\O(2^n)指数复杂度$
      • 空间复杂度
    • 分析
      • 基本操作：复杂度为O(1)，若为函数调用则将其整体时间复杂度代入计算
      • 加法规则（顺序复合）：若算法为多个部分的顺序复合，复杂性是着多个部分的复杂性之和，而由于忽略常量因子，则加法等价于求最大值
      • 乘法规则（循环结构）：若算法为一个循环，循环体执行$T_1(n)$次，每次循环体时间$T_2(n)$，则$T(n)=O(T_1(n)*T_2(n))$
      • 取最大原则（分支结构）：如果算法为条件分支，则复杂性为各分支结构中最大的
      • 递归算法的时间复杂度：
      • $T(n)=O(n^k)+a\times T(n/b)$
      • 若$a>b^k$则$T(n)=O(n^{\log_ba})$
      • 若$a=b^k$则$T(n)=O(n^k\log n)$
      • 若$a<b^k$则$T(n)=O(n^k）$
    • Python操作的时间开销
      • 基本算数运算为常量时间
      • 组合对象的复制和切片操作需要线性时间
      • list和tuple访问和修改元素为常量时间，而增删元素操作为线性时间
      • 构造新结构为线性时间
    • 程序实现和效率陷阱（某些步骤的具体实现可能使得原本可行的算法变得不可接受）

    • Python的内存对象表示
    • 对象标识(id)
    • 对象关联的表示
      • 利用元素存储位置隐式的表示（顺序表示）

      • 通常数据的表示为两种形式的混合

    • 变量与对象之间存在链接，而对象之间也存在链接

      抽象数据类型和Python类

  • 抽象数据类型：数据抽象，与函数所实现的计算抽象或过程抽象相对应。为目的的实现提供接口，而包装成的整个整体隐藏了其中的技术实现的细节。将对象的使用与其具体实现隔离。

    • ADT思想
    • 围绕一类数据定义程序模块
    • 接口与实现分离
    • 需要实现时从所用的编程语言中选取和使得机制

  • Python类

    • 头部，体
    • 属性
    • 以_开头的名字为内部的名字，无法在类之外访问；以__开头但不以其结尾的属性在类定义之外不能直接以此名称访问，从而保护内部数据
    • 类方法
    • 初始化方法init(self,arg1,arg2…)
    • 实例方法（操作对象的实例的方法）
    • 非实例方法（不依赖于类的对象，但是仅与类的实现的某一局部特性有关而不应定义为全局变量，称为静态方法，描述时需要在函数定义的头部之前加修饰符@staticmethod，且参数列表中不应该有self参数,可以通过从对象或类除法通过圆点形式调用，本质上说就是类里面定义的局部的普通函数）
    • 特殊方法（定义该类的某种算数运算，特殊方法名均以__开头和结尾，二元运算的参数为（self,another））
      • 四则运算：add,mul,floordiv,truediv,sub,mod
      • 比较:eq,lt,ne,le,gt,ge
      • 字符串化：str
    • 类定义和使用
    • 名字空间
    • 类中定义的变量和函数称为类的属性
    • 类对象的使用：属性访问和实例化
    • __doc__数据属性：类的文档串
    • 实例初始化的过程
      1. 创建类的对象
      2. 利用初始化函数为对象的属性赋值
      3. 返回新建的对象并通过赋值语句把新对象赋值给变量作为其约束值
    • 方法的定义和使用
      • 实例方法用圆点形式调用时，将调用该方法的对象约束到第一个形参self
      • 方法对象不同于函数对象，包含了两个成分：类中的函数定义生成的函数对象；调用时约束的一个实例对象
    • 说明
      • 创建了类的对象之后仍可以通过属性赋值的方式为类增加新属性，包括数据属性和函数属性
      • 若需要在一个方法函数中调用同一个类中的其他函数，需要利用self以属性描述的方式写法调用
      • 方法函数就是定义在类中的函数，可以访问全局名字空间，必要时可以用global或nonlocal声明
      • 内置函数isinstance(obj,cls)检查类和对象的关系
    • 类方法
      • 定义前加修饰符@classmethod,习惯以cls作为表示其调用类的参数，类方法执行时，调用它的类自动约束到方法的cls参数中，用于访问类的其他属性
    • 类定义的作用域规则
      • 函数中局部名字的作用域自动延伸到内部潜逃的作用域，而类定义中不然，在类的名字 并不延伸到类定义中嵌套的作用域，从而在内部嵌套的作用域中需要用基于类名的属性引用的方式
    • 私有变量
      • 约定以_开头的名字作为实例对象内部的东西，永远不从对象的外部访问
      • 以__开头的名字在类之外无法用属性访问的方式找到该名字
    • 类继承
    • 作用：基于已有的类定义新类，重复利用代码；建立类之间的继承关系，更好的组织和构造复杂的程序
    • 基类，派生类，继承关系，层次结构（不能出现循环结构）
    • 替换原理：在使用一个基类的实例对象的上下文中应当也能够使用其派生类
    • 内置函数issubclass(cls1,cls2)检验后者是否直接或间接的继承前者
    • 派生类的对象也是基类的对象
    • 方法查找
      • 从派生类的实例调用方法时，首先从实例对象所属的类开始，若没找到定义就在其基类中查找
      • 若在继承类中覆盖了原有的方法，则派生类中调用时就不会使用基类中的方法了
      • 动态约束，调用时基于self所表示的那个实例对象的类型去确定应该调用哪个函数（不同于静态约束的根据程序正文去确定被调用方法的方式）
    • 标准函数super()
      • super().m()从对象所属类的基类中开始查找方法函数
      • super(C,obj).m()从指定的类C开始查找函数属性m，找到后将C的一个实例obj作为m的self实参
    • Python异常
    • 异常处理：try,except
    • Python中的异常都是类，产生遗产个旧市生成相应的类的实例对象
    • 定义异常时从系统异常中选择一个合适的异常从中派生处自己的异常类
    • 若不需要定义的异常类执行特殊的操作，课利用pass填充函数体使结构完整
    • 异常的传播和捕获
      • 在try语句中查找处理异常的except语句
      • 若未找到则在外层的try语句中查找异常处理的方法
      • 若函数中不存在处理，则函数的执行异常终止，在函数的调用点引发异常，在该环境下查找异常处理的方法
      • 若某一布找到了异常处理的方法，则执行处理后继续执行try语句，否则层层退出直到最上层也没找到则终止程序
